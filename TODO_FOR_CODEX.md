# 河憩寮網站｜後續可交給 Codex 的任務清單

以下是建議用小步任務處理的項目。每次只挑一項做，不要一次大改整站。

## 高優先

### 1. 硬體販售資料容錯

目標：同商品ID的後續規格列，如果品名、分類、商品說明空白，可自動沿用上一列同商品ID資料。避免使用者在 Google Sheets 每一列都要重複貼完整資料。

注意：不可破壞目前多規格合併邏輯；空白欄位沿用規則應只套用在同商品ID內；售價、規格、狀態、規格備註不應被錯誤沿用。

### 2. 硬體販售手機版最終檢查

檢查商品圖是否完整顯示、規格名稱/備註/價格是否外溢、價格是否仍在卡片內、商品說明展開/收合是否正常、點規格名稱是否可切換主圖、點主圖是否可彈出大圖。

### 3. 後台欄位說明文件化

替每個 Google Sheets 分頁建立欄位說明，讓使用者之後新增商品時不容易填錯。

## 中優先

### 4. 加入硬體搜尋

可搜尋商品名稱、分類、規格、商品說明、規格備註。搜尋不應破壞分類收合邏輯。

### 5. 商品排序優化

同分類內依「排序」欄顯示；同商品ID的規格依「排序」或原始列順序顯示；若排序空白，保留原始順序。

### 6. 圖片載入容錯

圖片失敗時顯示 placeholder，不造成卡片高度崩壞，不顯示破圖圖示。

## 低優先

### 7. 管理用檢查工具

建立簡易檢查頁或 console 檢查：缺商品ID、售價空白、圖片網址非公開、同商品ID但品名不一致。

### 8. SEO 與分享圖優化

整理 meta title、meta description、Open Graph 圖片、favicon、社群分享文字。

### 9. README 整理

把 repo 現有 README 更新為目前 GitHub Pages 架構，不再保留過多舊版 Netlify 部署文字。

## 建議給 Codex 的任務格式

```text
請先閱讀 AGENTS.md、DATA_SCHEMA.md、CHANGELOG_HEQILIAO.md。

任務：只修改硬體販售區塊，讓同商品ID的後續規格列可沿用上一列的分類、品名、商品說明與圖片網址。

限制：
1. 不要修改 config.js。
2. 不要改動本週名單、造景介紹、設備介紹。
3. 不要改掉點規格名稱切換主圖與主圖彈出預覽功能。
4. 修改後請列出修改檔案與測試方式。
```
